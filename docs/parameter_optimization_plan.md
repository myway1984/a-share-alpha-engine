# 参数优化方案

## 目标

这份方案用于给当前 `qstrategy_v2` 主线策略建立一套可控、可解释、可复现的参数优化流程。

当前不建议直接做大规模黑箱搜索，而是先在“修正后的可信主线”上做分阶段优化，避免把历史 bug、市场风格切换和过拟合混在一起。

## 当前优化对象

当前主线以以下结构为基础：

- 修正后的股票池抽样逻辑
- `slowrebalance_3` 组合层
- 当前有效核心因子：
  - `one_month_reversal`
  - `turnover_20d`
  - `volatility_20d`
  - `ep`
  - `roe_ttm`
- 已剔除：
  - `price_volume_corr`
  - `sue`

## 为什么现在不直接用遗传算法

- 策略刚修完关键股票池 bug，先要保证基线稳定。
- `2024-11` 之后市场环境变化很大，直接做全历史全局最优，很容易优化出“旧环境最优、当前环境失效”的参数。
- 当前问题主要集中在组合层和交易节奏，参数空间还没有大到必须用遗传算法。
- 这套策略需要可解释性，尤其是持仓数、调仓频率、最小持有天数这些参数，不能只看回测最终数字。

## 优化方法顺序

建议按下面顺序推进：

1. 约束型小网格搜索
2. 贝叶斯优化
3. 遗传算法（可选，放最后）

### 阶段一：约束型小网格搜索

适用目标：

- 确定组合层的合理区间
- 快速排除明显差的参数组合
- 建立第一版“稳健参数带”

优先优化参数：

- `top_n`
  - 候选：`12 / 15 / 18 / 20`
- `buffer_rank`
  - 候选：`20 / 25 / 30 / 35`
- `rebalance_interval_trade_days`
  - 候选：`5 / 7 / 10`
- `min_holding_trade_days`
  - 候选：`5 / 10 / 15`
- `max_new_positions_per_rebalance`
  - 候选：`2 / 3 / 4`

约束原则：

- `buffer_rank >= top_n + 5`
- `min_holding_trade_days >= rebalance_interval_trade_days`
- `max_new_positions_per_rebalance <= top_n / 3`

### 阶段二：贝叶斯优化

适用目标：

- 在已知合理区间内进一步细化
- 联合优化组合层和少量因子权重
- 降低人工暴力枚举成本

建议纳入的额外参数：

- `one_month_reversal` 权重
- `turnover_20d` 权重
- `volatility_20d` 权重
- `ep` 权重
- `roe_ttm` 权重

建议范围：

- `one_month_reversal`: `0.5 ~ 1.5`
- `turnover_20d`: `0.25 ~ 1.25`
- `volatility_20d`: `0.5 ~ 1.5`
- `ep`: `0.5 ~ 1.75`
- `roe_ttm`: `0.0 ~ 1.25`

说明：

- 因子方向先不开放自动搜索，先固定方向。
- 先搜权重，不搜因子集合开关，避免参数空间突然失控。

### 阶段三：遗传算法

只有在下面条件同时满足时才考虑：

- 前两阶段已经证明参数空间确实复杂
- 组合层和因子权重联合优化仍然存在明显局部最优
- 我们已经建立了 walk-forward 验证流程

适合放进遗传算法的参数：

- 板块权重
- 行业约束参数
- 股票池配额参数
- 更长持有周期和更复杂换仓节奏

## 目标函数设计

不建议只用“最终收益率”做优化目标。

建议主目标函数采用加权评分：

`score = 年化收益 * 0.4 - 最大回撤 * 0.3 - 换手惩罚 * 0.15 + 稳定性得分 * 0.15`

更实际的落地方式：

- 主指标：
  - 区间收益
  - 最大回撤
  - 平均日换手
- 辅助指标：
  - 月度正收益占比
  - 分季度稳定性
  - 极端回撤次数

推荐筛选规则：

- 先过滤最大回撤过大的参数
- 再过滤换手过高的参数
- 最后在剩余集合中比较收益和稳定性

## 时间切分建议

当前最重要的是避免过拟合，所以必须做滚动验证，不建议只做一次性全样本优化。

建议分为三段：

- 训练段：`2019-01-01` 到 `2022-12-31`
- 验证段：`2025-01-01` 到 `2025-12-31`
- 当前环境检验段：`2024-11-01` 到 `2026-03-20`

原因：

- `2019-2022` 用来学习旧环境和多种市场风格
- `2025` 用来验证新一轮行情结构
- `2024-11` 之后单独拿出来，是因为市场叙事和结构已经显著变化

## Walk-Forward 验证

建议在正式优化时采用 walk-forward，而不是“一次训练、一次测试”。

建议节奏：

1. 用过去 `24` 个月数据优化参数
2. 用随后 `3` 个月做样本外验证
3. 滚动窗口重复
4. 汇总每个窗口的样本外表现

这样可以回答两个关键问题：

- 这个参数是不是只在某一段时期有效
- 参数是否会随着风格切换迅速失效

## 当前最值得先优化的部分

按优先级排序：

1. 组合层参数
   - `rebalance_interval_trade_days`
   - `min_holding_trade_days`
   - `max_new_positions_per_rebalance`
   - `top_n`
   - `buffer_rank`
2. 因子权重
   - 先只调当前保留的 5 个因子
3. 股票池参数
   - 例如板块抽样比例、流动性阈值

当前不建议优先动的：

- 因子方向自动搜索
- 行业趋势过滤参数
- 太复杂的板块专属模型

## 第一轮推荐实验

建议第一轮只做组合层小网格：

- `top_n`: `12 / 15 / 18`
- `buffer_rank`: `20 / 25 / 30`
- `rebalance_interval_trade_days`: `5 / 7 / 10`
- `min_holding_trade_days`: `10 / 15`
- `max_new_positions_per_rebalance`: `2 / 3 / 4`

预期组合数：

- `3 * 3 * 3 * 2 * 3 = 162` 组

这个规模是可控的，而且信息量足够高。

建议直接通过现有 CLI 入口运行：

```bash
PYTHONPATH=src python3 -m qstrategy_v2.cli \
  --provider tushare \
  --start-date 2019-01-01 \
  --end-date 2022-12-31 \
  --exclude-factors price_volume_corr,sue \
  --optimize-grid-only \
  --grid-search-output-dir reports/multifactor_v2_grid_search_train \
  --grid-top-n-values 12,15,18 \
  --grid-buffer-rank-values 20,25,30 \
  --grid-rebalance-interval-values 5,7,10 \
  --grid-min-holding-trade-day-values 10,15 \
  --grid-max-new-position-values 2,3,4
```

输出文件：

- `latest_grid_search.md`
- `latest_grid_search.json`

## 第二轮推荐实验

在第一轮筛出前 `10` 组之后，再做贝叶斯优化：

- 固定组合层参数范围在前 `10` 组附近
- 开始加入 5 个核心因子权重
- 目标是找到兼顾 `2025` 和 `2024-11 之后` 的稳健版本

## 通过标准

参数组合要同时满足：

- 在训练段不是单纯最高收益，而是风险收益更平衡
- 在验证段仍然保持正向或至少显著优于基线
- 在 `2024-11-01` 到 `2026-03-20` 这段新环境下不明显失效

## 当前结论

- 这套策略现在已经到了“应该做正式调优”的阶段
- 但不该直接做遗传算法式大范围黑箱搜索
- 最合理的路径是：
  - 先小网格确定组合层稳健区间
  - 再做贝叶斯优化细化因子权重
  - 最后才考虑更复杂的全局优化
