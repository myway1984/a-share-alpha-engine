# Model Upgrade Roadmap V1

## 目标

这份文档用于正式回应外部量化专家对当前模型的诊断意见，并把结论沉淀成后续迭代路线。

目标不是“照单全收”，而是：

- 判断哪些问题真实存在
- 区分哪些建议合理、哪些已经过时或不适用
- 明确未来模型的主要迭代方向
- 给出是否需要重写的结论
- 给出数据补充方案和预算建议

## 一、总体结论

外部意见有价值，但更像是基于仓库表层结构和 README 的第一轮框架审视，不是对当前最新代码与最新产出的完整核查。

因此应当：

- `吸收` 数据源稳健性、风控缺失、样本外验证不足这三类核心问题
- `修正` “没有回测输出”“没有因子诊断能力”这类已经落后于当前仓库状态的判断
- `拒绝照搬` 那些过于静态、过于交易手册化的建议，例如简单硬止损、单一阈值清仓规则

一句话总结：

`当前策略不需要推倒重写，更适合在现有架构上分阶段进化。`

## 二、外部意见逐项判断

### 1. 数据源单一且稳定性不足

判断：

- `真实存在`

原因：

- 当前主数据仍以 `Tushare + Eastmoney` 为主
- 对研究和日终复盘够用
- 对长期稳定运行、分钟级执行验证和更接近实盘的场景仍偏脆弱

结论：

- 合理，应作为高优先级问题处理

### 2. 因子有效性验证缺失

判断：

- `部分成立`

原因：

- 当前仓库并不是完全没有诊断能力
- 已经有因子诊断与参数优化模块：
  - `src/qstrategy_v2/diagnostics.py`
  - `src/qstrategy_v2/optimizer.py`
- 但还没有完全形成统一、标准化、可重复产出的“因子体检报告体系”

结论：

- 合理，但表述偏重
- 应理解为“能力已经有雏形，但输出标准还不够正式”

### 3. 回测设计需强化防未来函数

判断：

- `真实存在`

原因：

- 当前已经考虑了交易日历、财报公告日、滑点和手续费
- 但仍缺一份正式的 PIT 数据审计清单
- 也缺统一的“执行假设说明书”

结论：

- 合理，而且很重要

### 4. 风险控制模块空白

判断：

- `基本成立`

原因：

- 组合层已有慢换仓、持有期、缓冲区等弱风控
- 但没有独立 `risk.py`
- 没有完整的组合级、市场级、数据级暂停和刹车规则

结论：

- 合理，应进入下一阶段重点开发

### 5. 过度拟合防护缺失

判断：

- `部分成立`

原因：

- 当前已经有参数优化路线和网格搜索
- 也已经明确提出 walk-forward 方向
- 但还没有把滚动训练-验证-检验流程完全产品化

结论：

- 合理，但不是“完全缺失”，而是“尚未形成正式主流程”

## 三、哪些建议合理，哪些不合理或过时

## 合理建议

- 增强主备数据源能力
- 建立统一因子诊断报告
- 强化 PIT 和执行假设说明
- 引入独立风控层
- 建立滚动样本外验证

## 不合理或不适合直接照搬的建议

### 1. 固定阈值硬止损

例如：

- 个股 `-8%` 硬止损
- 市场跌停比例超过阈值即清仓

问题：

- 更适合偏短线、事件驱动或高频模型
- 对当前这种低频截面组合策略，容易：
  - 抬高换手
  - 破坏组合逻辑
  - 在 A 股常见波动中频繁误伤

结论：

- 不建议直接使用
- 如果要做，应改为组合层和状态层的动态风控，而不是固定单点阈值

### 2. 只因“免费接口”就直接否定现有研究底座

问题：

- 对研究阶段来说，当前数据源仍然具备足够价值
- 真问题不在“是不是免费”，而在：
  - 是否稳定
  - 是否 PIT
  - 是否能支撑执行层验证

结论：

- 不能因为数据源不是机构级就否定当前模型
- 应按阶段升级，而不是一步到位替换全部底座

### 3. 把“工具链”当成“策略建议”

例如某些建议直接把具体 skill 或外部平台作为模型方向的一部分。

结论：

- 工具链是实现手段，不是策略结论
- 选型应服务于数据质量和执行目标，而不是反过来

## 四、最大的启发

这份外部意见最大的启发不是“换几个因子”，而是：

`当前模型已经从“能不能搭起来”跨到了“能不能被长期信任和持续运行”的阶段。`

这意味着后面最重要的，不再只是继续堆因子，而是补厚四层能力：

1. `验证层`
   - 因子诊断
   - 参数稳定性
   - 样本外和滚动验证

2. `风控层`
   - 组合级控制
   - 数据异常暂停
   - 极端行情应对

3. `执行层`
   - 更真实的成交假设
   - 分钟级执行偏差评估
   - 纸面组合与理论组合误差监控

4. `状态层`
   - 市场状态识别
   - 行业拥挤度
   - 风格切换下的权重调整

## 五、未来模型的迭代方向

## 方向 1：从统一截面模型，升级到状态感知模型

目标：

- 不再假设一套参数永远有效
- 根据市场状态动态切换因子权重或组合约束

重点可研究：

- 风格状态分类
- 行业轮动状态
- 波动率 regime
- 拥挤度和成交结构

## 方向 2：从研究回测，升级到执行回测

目标：

- 让回测更接近真实交易环境

重点可研究：

- 次日开盘执行
- 集合竞价成交假设
- 分钟数据回放
- 高价股整手约束
- 容量和冲击成本

## 方向 3：从组合约束，升级到正式风控层

目标：

- 让模型在异常环境下能自动降速、暂停或收缩风险

重点可研究：

- 组合回撤控制
- 行业集中度上限
- 市场波动驱动的仓位调节
- 数据异常 / 资讯异常暂停

## 方向 4：从手工诊断，升级到标准化评估流水线

目标：

- 每轮迭代都可以被结构化比较

重点可研究：

- 因子体检报告模板
- 参数热力图
- walk-forward 输出模板
- 回测与纸面盘偏差报告

## 六、要不要重写

结论：

- `不要重写`

理由：

- 当前架构本身是清晰的
- 问题主要集中在：
  - 验证层
  - 风控层
  - 执行层
  - 数据稳健性
- 这些问题都更适合在现有框架上演进

只有当未来出现下面情况时，才考虑局部重构：

- 数据模型不再能支持状态切换
- 回测和执行层严重耦合，难以扩展
- 风控层无法独立插入

当前阶段更优做法是：

- `分阶段进化`
- `局部增强`
- `小步验证`

## 七、分阶段进化路线图

## 阶段 1：可信度增强

目标：

- 把“已有能力”变成“正式能力”

重点任务：

- 统一因子诊断报告
- 建立 walk-forward 验证流程
- 输出参数敏感性热力图
- 统一主线参数和样本区间文档

优先级：

- `最高`

## 阶段 2：风控层独立化

目标：

- 把组合约束升级为独立风控框架

重点任务：

- 新增 `risk.py`
- 组合回撤阈值
- 行业 / 主题暴露上限
- 数据异常暂停机制
- 极端市场暂停机制

优先级：

- `高`

## 阶段 3：执行层增强

目标：

- 让回测更接近真实执行

重点任务：

- 开盘 / 集合竞价执行模型
- 分钟数据支持
- 纸面账户与理论组合偏差分析
- 容量与整手约束增强

优先级：

- `高`

## 阶段 4：状态层升级

目标：

- 把策略从单一静态模型升级成状态感知模型

重点任务：

- 市场 regime 识别
- 状态驱动权重切换
- 行业拥挤度与风格暴露约束
- 板块和风格自适应

优先级：

- `中高`

## 八、是否需要更多数据支持

结论：

- `需要`
- 但不是立刻“全量重买机构级数据”
- 应按阶段补

## 阶段 A：继续做日频研究与模拟盘

当前所需补充：

- 历史 ST / 风险警示状态
- 更完整的行业历史映射
- 更清晰的复权与分红送转口径

预算建议：

- `¥500 - 3000 / 年`

可行方案：

- 继续使用 `Tushare + Eastmoney`
- 如有必要，补更高档位的 Tushare 权限和分钟数据

## 阶段 B：开始做执行回测

新增数据需求：

- 分钟线
- 集合竞价
- 更细粒度成交和涨跌停约束
- 更可靠的 PIT 财务

预算建议：

- `¥5000 - 20000 / 年`

建议关注的数据源：

- `JQData / 聚宽`
- `RQData / RiceQuant`
- 更高档 Tushare 方案

说明：

- 这一级预算主要用于“更稳”和“更真实”，不是为了花钱买更大名气

## 阶段 C：准实盘和多人协作

新增数据需求：

- 主备双源
- 稳定分钟/逐笔
- 更强财务和公告 PIT
- 更完整行业与风格库

预算建议：

- `¥30000 - 100000+ / 年`

建议关注的数据源：

- `Wind`
- `Choice`
- `iFinD`
- `聚源`

说明：

- 当前阶段不建议直接上这一档

## 九、当前推荐的数据采购顺序

1. 先把 `Tushare` 权限和使用方式做到最稳
2. 如要做执行回测，再补分钟级数据
3. 选 `RQData` 或 `JQData` 作为第二数据源做交叉验证
4. 只有准备更接近生产和多人协作时，再考虑机构级数据库

## 十、最终建议

当前最优策略不是：

- 推倒重写
- 盲目换因子
- 直接上昂贵机构数据库

当前最优策略是：

- 在现有清晰架构上继续演化
- 先补验证层和风控层
- 再补执行层和状态层
- 数据源按研究 -> 执行 -> 准实盘三个层级逐步升级

一句话总结：

`这套模型的下一个关键节点，不是“再加一个新因子”，而是把它从“能跑的研究系统”升级成“可信、稳、可持续迭代的策略平台”。`
