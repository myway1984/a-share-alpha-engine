# Version Roadmap

## 定位

这份路线图解决两个问题：

- 当前仓库已经走到哪一步
- 接下来每个版本主要要解决什么问题

这里的版本不是严格语义化发布，而是按策略能力阶段来定义。

## V0

`研究原型期`

特点：

- 多因子框架刚搭起来
- 先验证数据链、回测链和最小策略闭环
- 重点是“能跑通”，不是“够稳健”

这一阶段主要完成了：

- `Tushare` 历史数据接入
- 基础股票池过滤
- 多因子计算
- 初版回测与报告输出

## V1

`可运行研究版`

特点：

- 开始有稳定主线参数
- 可以连续跑回测
- 可以输出每日复盘

这一阶段主要完成了：

- 板块感知与中性化
- 慢换仓组合层
- 参数优化器
- 日常复盘和纸面账户流程

当前仓库主线可理解为 `V1.x`。

## V1.1

`当前主线`

核心目标：

- 固定一个能持续迭代的研究基线
- 让策略、复盘、模拟跟踪处于统一工作流中

当前已经具备：

- `Top N + Buffer + 最小持有交易日 + 调仓节奏` 的组合控制
- 每日复盘草稿生成
- 50 万模拟账户跟踪
- 独立仓库管理

当前仍然存在的边界：

- 不同市场风格下稳健性还要继续验证
- 风险控制还偏研究型，不是实盘型
- 数据源降级逻辑还可以更严

## V1.2

`稳健性增强版`

建议优先做的内容：

- 双数据源自动降级
- 更完整的数据质量校验
- 更明确的风险暂停机制
- 更系统的样本外验证

目标：

- 让“研究盘”和“日常运行”更稳定
- 降低单日数据异常对结论的影响

## V1.3

`执行增强版`

建议做的内容：

- 更贴近实盘的成交假设
- 组合层风控参数独立化
- 更清晰的行业 / 风格暴露诊断
- 更严格的异常订单处理逻辑

目标：

- 从研究框架，推进到“更接近真实执行”的系统

## V2

`实盘验证准备版`

进入这个阶段前，建议至少满足：

- 主线参数经过新市场环境验证
- 模拟账户跟踪足够长
- 回测、复盘、模拟盘之间的口径基本统一

这个阶段建议新增：

- 正式的实盘账户接入层
- 风控白名单 / 黑名单
- 运行告警
- 执行审计日志

## 当前建议的迭代顺序

1. 先把 `V1.1` 做扎实
2. 再补 `V1.2` 的稳定性能力
3. 然后推进 `V1.3` 的执行增强
4. 最后再进入 `V2`

一句话总结：

`现在最重要的不是把功能铺得更大，而是把当前主线做得更稳、更可复现、更容易持续维护。`
