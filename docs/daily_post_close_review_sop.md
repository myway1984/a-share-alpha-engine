# A股多因子策略 V2 收盘后复盘与调仓 SOP

## 1. 目的

这份 SOP 解决的是一件很具体的事：

- 每个交易日收盘后，如何用同一套步骤复盘大盘、板块、持仓、候选股和资讯。
- 如何把“模型信号”和“资讯判断”结合起来，决定次一交易日是否调仓、调多少、调哪些。

这套流程默认服务于当前 `qstrategy_v2` 主线策略，重点不是盘中追单，而是做 `T` 日收盘后复盘、`T+1` 日执行。

## 2. 总原则

- 先看数据质量，再看行情，再看持仓，再看候选，再看资讯，最后才下调仓结论。
- 模型负责给出“排序和候选”，人工负责识别“资讯风险、突发事件、执行可行性”。
- 若数据异常、信号异常、执行条件异常，优先暂停，不强行交易。
- 复盘结论必须沉淀成文档，不接受“看过了但没留下记录”。

## 3. 每日时间顺序

建议把收盘后流程固定在以下窗口内：

1. `15:05-15:20` 更新数据与校验
2. `15:20-15:40` 复盘大盘与板块
3. `15:40-16:10` 复盘现有持仓
4. `16:10-16:35` 审核模型候选与拟调仓名单
5. `16:35-17:00` 检索个股与市场资讯
6. `17:00-17:20` 输出次日调仓结论

## 4. 复盘输入

每天收盘后至少准备下面五类输入：

- 市场数据
  - 主要指数涨跌、成交额、涨跌家数、涨停跌停数量、北向资金、风格指数
- 板块数据
  - 行业涨跌幅、成交额变化、板块强弱轮动、当日最强与最弱方向
- 组合数据
  - 当前持仓、当日收益、相对收益、个股涨跌、调仓限制状态
- 模型数据
  - 最新股票池、最新排名、最新目标持仓、最新调仓清单
- 资讯数据
  - 宏观、政策、行业、持仓个股、拟买入个股的公告、新闻、研报、异动原因

## 5. 第一步：数据更新与校验

先完成数据层动作，再进入复盘。

必做检查：

- 当日交易日历已更新
- 全市场日线、估值、换手、停牌、涨跌停价已更新
- 股票池数量无异常跳变
- 主板 / 创业板 / 科创板数量结构未异常失真
- 关键字段无大面积缺失

若出现以下情况，当天不进入正常调仓：

- 数据未更新完整
- 停牌或涨跌停字段明显异常
- 股票池数量或板块结构明显失真
- 模型输出为空或出现大面积重复异常

处理规则：

- 先标记为 `数据异常日`
- 不执行新增仓位
- 如已有持仓风险极大，只允许人工风控卖出，不做模型调仓

## 6. 第二步：复盘大盘与板块

目标是先判断当天市场处在什么环境，而不是一上来就盯单只股票。

必须回答 5 个问题：

1. 大盘今天是普涨、普跌，还是结构分化？
2. 量能是放大、缩小，还是无效波动？
3. 强势方向集中在哪些行业和主题？
4. 弱势方向集中在哪些行业和主题？
5. 当前风格更偏大盘、成长、价值，还是题材炒作？

建议重点记录：

- 上证、深成、创业板、科创 50 的表现
- 全市场成交额与前一日对比
- 涨跌家数和涨停跌停数量
- 行业涨跌幅前 5 和后 5
- 是否存在一致性极强的主线叙事

这一步的目的不是直接决定买卖，而是给后面的持仓和候选审核提供“环境背景”。

## 7. 第三步：复盘现有持仓

对当前每一只持仓都要回答三个问题：

1. 它今天为什么涨或跌？
2. 它的核心逻辑还在不在？
3. 它是否进入了“该减、该卖、该继续拿”的状态？

逐只持仓至少检查这些内容：

- 当日涨跌幅、成交额、换手率是否异常
- 是否有公告、业绩预告、监管、减持、停牌、重大事项
- 是否出现放量长阴、跌停、冲高回落、连续缩量等异常形态
- 是否仍然在模型目标组合内
- 是否已经跌破缓冲区、且满足最小持有交易日约束

建议把持仓分成 4 类：

- `继续持有`
  模型仍支持，资讯无明显风险，执行上也无障碍。
- `继续观察`
  模型分数边缘化，但没有明确信号和资讯风险，不急于当天换出。
- `准备减仓/卖出`
  模型已不支持，或基本面/事件逻辑明显受损。
- `强制风险处理`
  出现重大负面公告、监管、流动性问题、板块逻辑崩塌等情况，允许人工优先处理。

## 8. 第四步：审核模型候选与拟调仓名单

这一步的目标是分清楚：

- 哪些是模型想继续持有的核心股票
- 哪些是模型新想换入的股票
- 哪些换仓是“有必要”，哪些只是“分数小幅波动”

重点看 4 类输出：

- `latest_rankings`
- `latest_orders`
- 当前持仓与目标持仓差集
- 候选股的因子结构

审核顺序：

1. 看模型前 `15` 名目标持仓
2. 看前 `5` 名是否集中在同一行业或同一风格
3. 看今天是否真的触发调仓
4. 看拟换入股票是不是仅因为短期噪声上升
5. 看拟换出股票是不是只是暂时回落但逻辑未破坏

对每一只拟换入股票，至少说明：

- 属于哪个板块和行业
- 哪几个因子在拉分
- 近期是否有明显利空或利好资讯
- 次日是否具备可执行性

## 9. 第五步：检索资讯与公告

这一步是每天收盘后必须做的“人工覆盖层”。

建议分 3 组检索：

- 市场级
  - 今日大盘异动原因
  - 当日最强板块和最弱板块原因
  - 宏观政策、行业政策、监管消息
- 持仓级
  - 当前 `15` 只持仓的公告、新闻、研报、龙虎榜、异动解读
- 候选级
  - 拟换入前 `5` 到 `10` 只股票的公告、新闻、业绩、异动原因

建议的数据源分工：

- `Tushare`
  - 用于日线、估值、换手、停牌、财务等结构化数据
- 东方财富金融数据 skill
  - 用于补充行情、板块、主力资金、估值、个股财务快查
- 东方财富资讯搜索 skill
  - 用于检索公告、新闻、研报、事件解读

资讯审核时，重点排查这些风险：

- 业绩大幅低于预期
- 监管问询、立案、处罚
- 大股东减持、解禁、质押风险
- 重大诉讼、停牌、资产重组失败
- 板块主线明显转弱
- 涨停后一字、缩量、无法成交风险

建议直接使用的检索问句：

- 市场级
  - `今日A股大盘异动原因`
  - `今日A股最强板块原因`
  - `今日A股最弱板块原因`
  - `今日A股盘后政策解读`
- 持仓级
  - `<股票名称> 最新公告`
  - `<股票名称> 最新资讯`
  - `<股票名称> 今日异动原因`
  - `<股票名称> 最新研报`
- 候选级
  - `<股票名称> 是否有利空`
  - `<股票名称> 业绩预告`
  - `<股票名称> 龙虎榜`
  - `<股票名称> 板块逻辑`

## 10. 第六步：输出调仓结论

复盘的结果必须落到一个明确结论，只能四选一：

- `不调仓`
  - 当前持仓继续保留，次日无主动操作
- `小幅调仓`
  - 只替换 `1-3` 只，符合当前慢换仓主线
- `风险减仓`
  - 不完全按模型，而是优先降低高风险暴露
- `暂停执行`
  - 当日数据、资讯或市场环境异常，次日不执行模型调仓

输出结论时，必须明确：

- 次日计划买入哪些股票
- 次日计划卖出哪些股票
- 每笔操作的原因
- 哪些候选只是观察，不立即执行
- 是否需要盘前再次确认公告或竞价状态

## 11. 调仓决策规则

建议把“模型结论”和“人工结论”合并成下面的判断顺序。

### 11.1 可以直接按模型执行的情况

- 数据正常
- 当天确实触发调仓
- 拟换入股票无重大负面资讯
- 拟换出股票不属于误杀型短期波动
- 次日不存在明显无法成交风险

### 11.2 需要人工复核后再执行的情况

- 拟换入股票当天刚大涨、涨停或异动明显
- 持仓股有重大公告待进一步确认
- 市场环境出现极端缩量或极端一致性
- 模型前几名过度集中在单一行业

### 11.3 应该覆盖模型、不执行换入的情况

- 股票出现重大利空、监管或财务异常
- 明显可能一字板买不到
- 流动性不足，实盘难以成交
- 题材已退潮，资讯面明显恶化，而模型尚未及时反映

## 12. 每日产出物

每天收盘后，至少沉淀以下产出：

- 一份大盘与板块复盘
- 一份持仓检查表
- 一份模型候选与拟调仓清单
- 一份资讯摘录
- 一份最终结论

建议文件结构固定为：

- `trade_date`
- `market_review`
- `portfolio_review`
- `candidate_review`
- `news_review`
- `action_plan`
- `risk_flags`

## 13. 当前主线下的建议默认规则

如果没有特别说明，当前建议沿用下面的默认执行约束：

- 目标持仓：`15`
- 调仓频率：每 `5` 个交易日一次
- 最小持有交易日：`10`
- 每次最多新增：`3`
- 若当天模型未触发调仓，则优先保持现有持仓

这意味着：

- 每天都要复盘
- 但不是每天都要交易
- 复盘的目的，是为真正需要交易的那天提前做好判断和准备

## 14. 一票否决项

以下情况出现任一条，都建议次日不按模型新增仓位：

- 数据质量异常
- 拟买入股票存在重大负面公告
- 拟买入股票高概率一字涨停无法成交
- 拟卖出股票跌停无法卖出，导致组合无法按计划再平衡
- 模型结果被极端单一主题挤占

## 15. 复盘流程的边界

这套流程解决的是“如何稳定复盘和出结论”，不是“保证每天都能调对仓”。

它的价值主要在三点：

- 把模型、资讯、执行连接起来
- 把每天的主观判断限制在固定范围内
- 把每一次调仓都变成可回溯、可复盘、可改进的决策

## 16. 复盘草稿生成命令

当前项目已经支持在跑完策略后，直接生成一份收盘后复盘草稿。

建议命令格式：

```bash
PYTHONPATH=src python3 -m qstrategy_v2.cli \
  --start-date 2026-01-01 \
  --end-date 2026-03-20 \
  --provider tushare \
  --top-n 15 \
  --buffer-rank 25 \
  --universe-limit 50 \
  --rebalance-interval-trade-days 5 \
  --min-holding-trade-days 10 \
  --max-new-positions-per-rebalance 3 \
  --generate-daily-review \
  --daily-review-output-dir reports/daily_reviews
```

输出内容包括：

- 常规回测摘要
- 最新持仓
- 最新排名
- 最新调仓指令
- 一份可直接继续填写的每日复盘草稿

如果后面确认了最终主线参数，可以把对应的因子权重和板块权重一并固化到命令里，做成每日固定任务。

## 17. 每日固定命令

如果你要执行“昨日信号 -> 今日模拟账户 -> 今日复盘草稿”这条固定流程，建议直接用这一条：

```bash
PYTHONPATH=src python3 -m qstrategy_v2.daily_workflow --trade-date 2026-03-23
```

以后日常使用时，只需要把日期替换成当天交易日：

```bash
PYTHONPATH=src python3 -m qstrategy_v2.daily_workflow --trade-date YYYY-MM-DD
```

这条命令当前默认会做 3 件事：

- 读取当前主线信号文件：
  - `reports/multifactor_v2_2026_ytd_slowrebalance_3_u50/latest_backtest.json`
- 生成并更新 `50万` 模拟账户
- 生成当天的复盘草稿

默认输出位置：

- 模拟账户：
  - `reports/paper_accounts/paper_500k.md`
  - `reports/paper_accounts/paper_500k.json`
- 每日复盘：
  - `reports/daily_reviews/YYYY-MM-DD_review.md`

如果后面你切换到新的主线报告，只要加一个参数覆盖信号文件即可：

```bash
PYTHONPATH=src python3 -m qstrategy_v2.daily_workflow \
  --trade-date YYYY-MM-DD \
  --signal-report path/to/latest_backtest.json
```
